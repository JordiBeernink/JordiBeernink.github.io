Good news, everyone!

February 9, 2017 Programming


If you can read this, it means I have succesfully gotten Hexo up and running. Great! Hexo is a node.js framework that does most of the things that Jekyll does, but uses node’s framework.

Hexo offers:

  • Good speed
  • Command line interface
  • lots of templates and plugins
  • It’s easy to migrate to Hexo from wordpress/Jekyll/Joomla etc

Hexo generates the static content via the command line and takes as input markdown (.md) and YAML (.yml) files. Hardly any coding is needed but I usually costumize these things a fair bit - for that I temper with a fair bit of javascript in the “embedded JavaScript” (.ejs) files.

A rough workflow for getting Hexo up and running is as following:

  • Install node.js and git
  • Use node’s package manager (npm) to fetch hexo-cli and hexo-server
  • Use hexo init <projectname> in your command terminal
  • Watch the magic happen
  • In your new blog folder, use npm install - this should install all the node modules (specified in package.json) required for your blog.
  • Blogs need posts. Blogs love posts. Lets get that blog a post: hexo new "<titleOfBlogPost>"
  • edit the markdown post (in /source/) with your editor of choice
  • run hexo clean (clears your public folder) and hexo generate to turn that lovely markdown into html
  • inspect your new website with hexo server

Since the static website is all contained in the /public/ folder, you can push that to your gh-pages branch.

Alternatively you could install hexo-deployer-git to have hexo build and deploy automatically to your repo from the command line.
